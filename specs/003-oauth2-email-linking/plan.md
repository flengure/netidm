# Implementation Plan: OAuth2 Email-Based Account Linking

**Branch**: `003-oauth2-email-linking` | **Date**: 2026-04-17 | **Spec**: [spec.md](spec.md)

## Summary

When a user logs in via GitHub or Google OAuth2 and their verified email matches an existing local Person account's primary email, the system links the OAuth2 identity to the existing account instead of JIT-creating a duplicate. Linking is controlled by a per-provider boolean setting that overrides a global domain-level default. Both default to off.

### Step 5 — `find_and_link_account_by_email`

**File**: `server/lib/src/idm/server.rs`

New method on `IdmServerProxyWriteV1` (or equivalent write context), placed near `find_account_by_oauth2_provider_and_user_id`:

```rust
pub fn find_and_link_account_by_email(
    &mut self,
    provider_uuid: Uuid,
    claims: &ExternalUserClaims,
) -> Result<Option<Account>, OperationError> {
    // Guard: no email
    let email = match &claims.email {
        Some(e) => e,
        None => return Ok(None),
    };

    // Guard: explicitly unverified (None = treat as verified, e.g. GitHub)
    if claims.email_verified == Some(false) {
        return Ok(None);
    }

    // Search by primary email (indexed equality)
    let filter = filter_all!(f_eq(
        Attribute::Mail,
        PartialValue::EmailAddress(email.clone()),
    ));
    let entries = self.qs_write.internal_search(filter)?;

    // Guard: ambiguous or no match
    if entries.len() != 1 {
        return Ok(None);
    }
    let entry = entries.into_iter().next().expect("len==1");

    // Guard: already linked to a provider
    if entry.get_ava_single_refer(Attribute::OAuth2AccountProvider).is_some() {
        return Ok(None);
    }

    // Link: add OAuth2Account class + three attributes
    let cred_id = Uuid::new_v4();
    let target_uuid = entry.get_uuid();
    let modlist = ModifyList::new_list(vec![
        Modify::Present(Attribute::Class, EntryClass::OAuth2Account.into()),
        Modify::Present(Attribute::OAuth2AccountProvider, Value::Refer(provider_uuid)),
        Modify::Present(
            Attribute::OAuth2AccountUniqueUserId,
            Value::new_utf8(claims.sub.clone()),
        ),
        Modify::Present(Attribute::OAuth2AccountCredentialUuid, Value::Uuid(cred_id)),
    ]);
    self.qs_write.internal_modify(
        &filter_all!(f_eq(Attribute::Uuid, PartialValue::Uuid(target_uuid))),
        &modlist,
    )?;

    // Reload and return
    let updated = self
        .qs_write
        .internal_search_uuid(target_uuid)?;
    Account::try_from_entry_rw(&updated, &mut self.qs_write)
        .map(Some)
        .map_err(|e| e)
}
```

### Step 7 — Tests

**File**: `server/testkit/` or inline unit tests in `server.rs`

Required tests (per constitution Principle III):

1. **Email match + verified → links, no duplicate** — provision local Person with email; complete OAuth2 flow with same verified email; assert one account, oauth2 attributes set.
2. **Email match + `email_verified = false` → no link, JIT creates new** — same setup, unverified email; assert two accounts created.
3. **Email match + provider disabled → no link** — `email_link_accounts = false` on provider; assert two accounts.
4. **Email match + global disabled + provider not set → no link** — global off, no per-provider; assert two accounts.
5. **Email match + global disabled + provider enabled → links** — global off, per-provider true; assert one account.
6. **Ambiguous match (2 accounts same email) → no link, JIT** — two local accounts same email; assert third account created.
7. **Already-linked account → no overwrite** — existing linked account; second provider attempt; assert original link preserved.
8. **Subsequent login uses (provider, sub) path** — after linking, second login skips email search.

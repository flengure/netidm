# Implementation Plan: Generic OIDC Upstream Connector

**Branch**: `006-oidc-connector` | **Date**: 2026-04-18 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/006-oidc-connector/spec.md`

## Summary

Add a generic OIDC upstream connector that auto-discovers all provider endpoints from an issuer URL, cryptographically verifies `id_token` JWTs using the provider's JWKS, and exposes a `create-oidc` CLI command. Extends the existing `OAuth2ClientProvider` with two new optional fields (`issuer`, `jwks_uri`), follows the existing `AuthExternal` continuation pattern for JWKS fetching in `server/core`, and introduces two new schema attributes in a DL21 migration. Zero new external crate dependencies.

## Technical Context

**Language/Version**: Rust stable (see `rust-toolchain.toml`)
**Primary Dependencies**: `compact_jwt` 0.5.6 (workspace, already present — provides `OidcUnverified`, `JwkKeySet`, `JwsEs256Verifier`, `JwsRs256Verifier`); `reqwest` (workspace, already in `server/core` and `libs/client`); `serde_json` (already present)
**New Dependencies**: None
**Storage**: Netidm MVCC entry database — DL21 migration adds two URL-type `systemmay` attributes to `EntryClass::OAuth2Client`
**Testing**: `cargo test` via `server/testkit` integration test infrastructure (real in-process netidmd)
**Target Platform**: Linux server (same as rest of netidm)
**Project Type**: Library + HTTP service + CLI tool
**Performance Goals**: JWKS fetch adds one HTTP round-trip per login where `id_token` is present; acceptable for interactive login
**Constraints**: `server/lib` must remain network-free (no reqwest); JWKS fetch handled by `server/core` via `AuthExternal` continuation pattern

### Layer 6: JWKS Fetch in server/core (`server/core/src/https/views/login.rs`)

Add match arm to the `AuthState::External` loop:
```rust
AuthExternal::OAuth2JwksRequest { jwks_url, id_token, access_token } => {
    let claims_body = verify_oidc_id_token(state, &jwks_url, &id_token).await
        .map_err(|_| OperationError::NotAuthenticated)?;
    let auth_cred = AuthCredential::OAuth2JwksTokenResponse { claims_body };
    let inter = state.qe_r_ref.handle_auth(Some(sessionid), AuthStep::Cred(auth_cred), ...).await?;
    auth_state = inter.state;
    continue;
}
```

Add `async fn verify_oidc_id_token(state: &ServerState, jwks_url: &Url, id_token: &str) -> Result<String, OperationError>`:
1. GET `jwks_url` using `state`'s reqwest client → `JwkKeySet`
2. `OidcUnverified::from_str(id_token)?`
3. Find `Jwk` by `kid` + algorithm
4. If not found: re-fetch JWKS once (key rotation), retry key lookup
5. `JwsEs256Verifier::try_from(&jwk)?.verify(&unverified)?.verify_exp(now)?`  (or Rs256)
6. Serialize token claims to JSON: `serde_json::to_string(&token.extra.unwrap_or_default())`  + inject `sub` field
7. Return JSON string

## v0.1.29 — Operator documentation, config wiring, and connector examples

This release focuses on making netidm production-ready for operators migrating from dex. It ships
the Phase E documentation layer, wires three previously-deferred config fields, adds ready-to-run
connector example scripts, and fixes a WireGuard peer resolution edge case.

### Operator documentation (Phase E)

- **Connector book pages** — each of the 14 upstream connectors (GitHub, Google, Microsoft, LDAP,
  SAML, LinkedIn, OpenShift, GitLab, Bitbucket, AuthProxy, Gitea, Keystone, Atlassian Crowd,
  Generic OIDC) now has a dedicated configuration reference page in the operator book.
- **Dex migration guide** — step-by-step guide for translating a dex `config.yaml` into the
  equivalent netidm configuration, including connector mapping tables and connector CRUD API
  examples.
- **Config section documentation** — `[oauth2]`, `[web]`, `[expiry]`, and `[forward_auth]`
  sections are documented with all accepted fields, types, defaults, and examples.

### Config fields wired

Three fields that were parsed but previously ignored are now fully wired through to the runtime:

- `oauth2.pkce_enforce` — when `true`, PKCE (RFC 7636) is required for all authorization code
  flows regardless of client type.
- `oauth2.always_show_login_screen` — forces the SSO connector picker to appear even when only
  one upstream connector is configured.
- `oauth2.skip_approval_screen` — bypasses the consent screen globally; consent tokens are
  auto-permitted without user interaction.

### Connector example scripts

`scripts/connectors/` contains ready-to-use shell scripts that call the connector CRUD API
(introduced in v0.1.28) to create each supported connector type. Each script includes a
commented-out example of every supported configuration field.

### Bug fix

- **WireGuard peer list user resolution** — `GET /v1/wg/peers` previously returned unresolved
  UUIDs for peer entries whose linked user account had been renamed. Peer list now resolves
  display names correctly across renames.
